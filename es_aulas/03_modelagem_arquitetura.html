Linguagem de Modelagem Unificada (UML)

Padrão para Visualização, Especificação e Documentação

O que é UML?

  • Linguagem padrão para visualização, especificação e documentação
  • Notação gráfica com semântica definida
  • Independente de linguagem de programação

Principais Diagramas UML

  • Estruturais: Classes, Objetos, Componentes
  • Comportamentais: Casos de Uso, Sequência, Atividades

Diagrama de Classes

Modelando a Estrutura Estática do Sistema

Elementos do Diagrama

  • Classe: atributos e operações
  • Associações: relacionamento entre classes
  • Multiplicidade: quantidade de instâncias
  • Generalização: herança

Relacionamentos

  • Associação
  • Agregação e Composição
  • Dependência
  • Generalização

Arquitetura de Software

Estruturando Sistemas de Software

O que é?

  • Estrutura fundamental de um sistema
  • Define componentes e suas interações
  • Influencia qualidades como desempenho e segurança

Benefícios

  • Compreensão do sistema
  • Reutilização de componentes
  • Manutenibilidade e evolução facilitadas

Engenharia de Software - Aula 3

Padrões Arquiteturais Comuns

Arquitetura em Camadas

  • Organiza o sistema por funcionalidades
  • Camadas: Apresentação, Negócio, Persistência
  • Vantagem: Separação de preocupações

Model-View-Controller (MVC)

  • Separa lógica de negócio e interface
  • Modelo: Dados; View: Interface; Controller: Lógica de controle
  • Vantagem: Manutenção paralela

Cliente-Servidor

  • Divide em provedores e consumidores de serviços
  • Vantagem: Centralização e segurança

Arquitetura Orientada a Serviços (SOA)

  • Sistema composto por serviços independentes
  • Vantagens: Reuso, interoperabilidade, escalabilidade
