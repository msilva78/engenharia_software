Qualidade de Código

O que significa escrever um "bom" código?

Características Chave

  • Legibilidade: Fácil de ler e entender.
  • Manutenibilidade: Simples de modificar e corrigir.
  • Eficiência: Consome menos recursos (memória, CPU).
  • Escalabilidade: Suporta crescimento e novos recursos.

Por que é Importante?

Um código de alta qualidade é a base para um software que pode evoluir, ser corrigido rapidamente e ser mantido por uma equipe.

Boas Práticas de Programação

Diretrizes para um Código de Qualidade

Recomendações

  • Use nomes de variáveis e funções descritivos.
  • Adicione comentários para explicar lógicas complexas.
  • Siga um padrão de codificação (style guide).
  • Refatore o código regularmente.

Padrões de Projeto

Soluções reutilizáveis para problemas comuns no projeto de software, como o padrão Singleton ou Factory.

Teste de Software

A arte de encontrar bugs

O que é?

O processo de execução de um sistema com a intenção de encontrar erros.

  • Garante que os requisitos foram atendidos.
  • Identifica e reduz riscos de falhas.
  • Aumenta a confiança no produto.

Objetivos

  • Verificar se o software funciona como esperado.
  • Garantir a qualidade e a conformidade.
  • Fornecer informações sobre a qualidade do produto.

Tipos de Teste

A Pirâmide de Testes

Teste Unitário

  • O quê: Componentes ou funções individuais.
  • Quem: Desenvolvedor.
  • Vantagem: Encontra bugs cedo, fácil de automatizar.

Teste de Integração

  • O quê: Interações entre módulos.
  • Quem: Desenvolvedores/QA.
  • Vantagem: Garante a comunicação correta.

Engenharia de Software - Aula 4

Tipos de Teste

Testando o Sistema Completo

Teste de Sistema

  • O quê: O sistema como um todo.
  • Quem: Equipe de QA.
  • Vantagem: Simula o uso real.

Teste de Aceitação

  • O quê: Se o sistema atende aos requisitos do cliente.
  • Quem: Cliente/Usuário.
  • Vantagem: Valida o produto com base nas necessidades.
